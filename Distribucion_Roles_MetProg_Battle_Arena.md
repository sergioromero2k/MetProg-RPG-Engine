# Distribución de Roles y Perfiles

## Opción 1: Grupo de 4 Personas (Ideal)
Esta es la configuración completa solicitada en el enunciado.

- **Jefe de Proyecto + Analista Funcional**: Se encarga de la plantilla de análisis, la gestión en Trello y es el único que realiza la entrega en el Aula Virtual.
  
- **Analista Programador**: Responsable de los diagramas UML (clases, secuencia, estados, etc.) en PlantText.
  
- **Ingeniero de Desarrollo**: Lidera la codificación en Java JDK 21, estableciendo la estructura de herencia de los personajes y el sistema de persistencia.
  
- **QA (Quality Assurance)**: Desarrolla todas las pruebas unitarias necesarias para verificar el sistema y redacta el documento de pruebas.

## Opción 2: Grupo de 3 Personas
Dado que el mínimo permitido son 3 personas, se deben fusionar responsabilidades manteniendo los perfiles obligatorios:

- **Jefe de Proyecto + Analista Funcional**: Mantiene sus funciones de gestión, entrega y definición de requisitos.
  
- **Ingeniero de Desarrollo + Analista Programador**: Esta persona asume toda la carga técnica, desde el diseño de los diagramas UML hasta la implementación del código en Java.
  
- **QA (Quality Assurance)**: Se encarga de las pruebas unitarias y de asegurar que los requisitos (como el formato del número de registro LNNLL) se cumplan estrictamente.

## Opción 3: Grupo de 2 Personas (No recomendado/Excepcional)
**Nota**: El enunciado especifica grupos de 3 o 4 personas. Si sois 2, deberíais consultar con el profesor, pero la carga se repartiría así:

- **Perfil A (Gestión y Diseño)**: Jefe de Proyecto, Analista Funcional y Analista Programador. Se encarga de toda la documentación previa y diagramas.
  
- **Perfil B (Ejecución y Control)**: Ingeniero de Desarrollo y QA. Se encarga de que el código compile, ejecute, sea persistente y pase las pruebas unitarias.

## Responsabilidades Compartidas
Independientemente del número de personas, todos deben dominar:

- **Lógica de Combate**: Entender el cálculo de éxitos (dados 5 y 6) y el potencial de ataque/defensa.
  
- **Jerarquía de Personajes**: Diferenciar el uso de Sangre (Vampiros), Rabia (Licántropos) y Voluntad (Cazadores).
  
- **Uso de GitHub**: Todos deben ser colaboradores del repositorio para que los profesores puedan evaluar la actividad individual.
